## 문서 5 · 심층 분석 및 인터뷰 답변

### 파이프라인 설계 순서

기획(브랜드·핵심 메시지·씬 구조 확정) → 생성(이미지 우선 확정 후 비디오 변환) → 검수(톤앤매너·비율·해상도 일치 확인) → 편집/통합(CapCut 컷 편집·자막·오디오 레벨) 순서로 설계했습니다. 이미지를 먼저 전체 씬에서 확정한 이유는, 비디오 변환(Runway)이 이미지 생성보다 크레딧 소모가 크기 때문에 톤이 안 맞는 이미지를 비디오로 넘기면 크레딧 낭비가 커지기 때문입니다. 검수 단계에서는 3개 씬의 색감(회색→딥그린 톤 전환)이 자연스럽게 이어지는지, 화면 비율이 16:9로 통일됐는지를 편집 전에 먼저 확인했고, 이 단계를 건너뛰지 않은 것이 STEP 6의 프롬프트 재수정으로 이어졌습니다.

### 결과 파일 네이밍/정리 규칙

`{유형}_{씬번호}_{내용키워드}.{확장자}` 형식으로 통일했습니다. 유형은 img/video/audio, 씬번호는 scene01~03, 내용키워드는 영어 소문자와 언더스코어만 사용(예: img_scene02_tumbler.png)해서 어떤 도구로 만든 무슨 씬인지 파일명만 보고 바로 알 수 있게 했습니다. 최종 산출물은 브랜드명+용도+비율(EcoWave_Brand_Campaign_16x9.mp4)로 구분해, 중간 산출물과 최종 제출본이 섞이지 않도록 했습니다.

### Text-to-Image vs Image-to-Video 선택 기준

Text-to-Image(Midjourney)는 정지된 한 장의 구도·색감·질감을 빠르고 저렴하게 확정하는 데 적합하고, 브랜드 룩앤필을 씬마다 고정하기 쉽습니다. Image-to-Video(Runway Gen-2)는 이미 확정된 이미지 위에 모션만 얹기 때문에 화풍이 유지된 채로 움직임만 통제할 수 있는 반면, Text-to-Video는 이미지 단계 없이 바로 영상을 생성해 편리하지만 씬마다 색감·구도가 미세하게 흔들릴 위험이 큽니다. 이 프로젝트는 브랜드 톤 일관성이 핵심 평가 기준이라, 이미지로 먼저 톤을 고정한 뒤 Image-to-Video로 넘기는 방식을 선택했습니다.

### 도구 선택 우선순위: 품질/제어/속도/비용

**제어(Control)**를 최우선으로 뒀습니다. 이유는 이 캠페인의 핵심이 "세련되고 일관된 톤"이라, 결과물이 매번 다르게 나오면 브랜드 경험이 깨지기 때문입니다. Midjourney는 `--ar`, 재질·조명 키워드로 세부 제어가 가능해 선택했고, 속도나 비용이 더 유리한 다른 도구 대신 제어력을 우선했습니다. 오디오는 ElevenLabs가 감정 톤과 발화 속도를 세밀하게 조정할 수 있어(제어) Suno보다 내레이션에 더 적합했고, Suno는 반대로 짧은 징글 생성 속도가 중요해서 엔딩 컷에만 사용했습니다.

### 프롬프트 A→B 변화의 원인

수정 전 프롬프트는 "recycled plastic tumbler"처럼 소재 자체를 지시해, 모델이 재활용 플라스틱의 거친 반사광(유광 표면)을 기본값으로 렌더링했습니다. 수정 후에는 `matte texture`, `high-end studio lighting`처럼 표면의 반사 특성과 조명 조건을 직접 지정해, 모델이 정반사(글로시) 대신 확산반사(매트) 쪽으로 렌더링 가중치를 옮기도록 유도했습니다. 즉 소재명만으로는 표면 질감이 모델의 기본 가정에 맡겨지지만, 재질·조명 키워드를 명시하면 그 가정을 직접 덮어쓸 수 있다는 것이 변화의 원인입니다.

### 심층 인터뷰 답변

**도구 간 결과물 통합 보정 기준:** Midjourney·Runway·ElevenLabs·Suno는 각각 색온도와 라우드니스 기준이 달라, CapCut에서 전 씬에 동일한 색보정 프리셋(LUT)을 적용해 회색→그린 색조 전환이 부자연스럽지 않게 통일했고, 오디오는 라우드니스 정규화로 내레이션과 BGM의 음량 편차를 맞췄습니다.

**동일 스토리보드를 다른 도구로 적용했을 때 예상되는 차이:** Runway 대신 Luma Dream Machine을 썼다면 모션이 더 유기적으로 부드럽지만, 딥그린 색 재현이 미세하게 달라져 씬 2~3 사이 색보정 작업이 추가로 필요했을 것으로 예상합니다.

**크레딧 부족 시 구체적 변경 전략:** 3씬을 유지하기 어려울 경우 씬 1(문제 제기)과 씬 2(해결)를 하나로 합치고, 씬 3(브랜드 인지)만 남기는 2씬 구성으로 축소하거나, 비디오 변환 없이 정지 이미지에 패닝/줌(Ken Burns 효과)만 적용해 크레딧 소모가 큰 Image-to-Video 단계를 생략하는 방안을 준비했습니다.

**60초 분량을 15초로 줄여야 할 경우 메시지 재구성 방안:** 60초 버전이 문제 제기-해결-사용 장면-브랜드 인지 4단계로 구성되어 있었다면, 15초로 줄일 때는 중간의 '일상 사용 장면'처럼 설득을 돕지만 핵심 메시지 전달에 필수는 아닌 구간부터 먼저 제거합니다. 남기는 3단계(문제 제기 3초 → 해결/제품 강조 6초 → 브랜드 인지 3~5초)는 지금의 9초 버전 구조와 동일하게 유지하되, 내레이션 문장 수를 줄이고 한 문장에 정보를 압축해서("바다가 삼켜버린 미래, 이제 에코웨이브가 돌려놓습니다"처럼) 각 씬이 짧아져도 메시지 전달력이 떨어지지 않도록 재구성합니다. 즉 씬을 쳐내는 기준은 '분량'이 아니라 '핵심 메시지 기여도'이며, 브랜드 인지 장치(로고·슬로건·CTA)가 들어가는 마지막 구간은 축소 대상에서 항상 제외합니다.
